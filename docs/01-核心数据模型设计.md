# tmc-common 核心数据模型设计

## 一、模块定位

`tmc-common` 被 SDK 和服务端同时依赖，包含**数据协议**、**常量**和**配置接口**，不包含任何业务逻辑。

## 二、数据模型

### 1. KeyAccessEvent — 访问事件协议

对应文章 **4-2 数据收集**，SDK 通过 rsyslog → Kafka 上报给服务端。

```java
public class KeyAccessEvent {
    private String appName;      // 业务应用名称
    private String uniqueKey;    // 缓存的 key
    private long sendTime;       // 事件发生时间戳
    private int weight;          // 访问权值（普通请求=1）
}
```

服务端内存存储结构：`Map<String, Map<String, LongAdder>>` → `Map<appName, Map<uniqueKey, 热度>>`

### 2. HotKeyEntry — 热点 Key 推送协议

服务端 → etcd → SDK，通知 SDK 将哪些 key 加入本地缓存。

```java
public class HotKeyEntry {
    private String appName;      // 所属应用
    private String uniqueKey;    // 热点 key
    private long heat;           // 当前滑窗总热度
    private long expireAt;       // 热点过期时间
}
```

### 3. InvalidateEvent — 失效广播协议

写入端（`set/del/expire`）→ etcd → 集群其他 SDK 节点，保证集群最终一致。

```java
public class InvalidateEvent {
    private String appName;
    private String uniqueKey;
    private InvalidateEventType eventType;  // SET / DEL / EXPIRE
    private long timestamp;
}
```

## 三、核心常量

| 常量 | 值 | 来源 |
|---|---|---|
| `TIME_WHEEL_SLOTS` | 10 | 文章 4-3-1：时间轮共 10 个时间片 |
| `SLOT_INTERVAL_SECONDS` | 3 | 每 3 秒一个周期 |
| `WINDOW_SECONDS` | 30 | 10 片 × 3s = 30s 滑动窗口 |
| `LOCAL_CACHE_MAX_BYTES` | 64MB | 文章 3-2-3：本地缓存上限 64MB（LRU） |
| `DEFAULT_HOT_THRESHOLD` | 100 | 默认热点阈值 |
| `DEFAULT_TOP_N` | 100 | 默认 TopN 数量 |

## 四、etcd 路径规划

```
/tmc/hotkeys/{appName}     → 热点 key 列表推送
/tmc/invalidate/{appName}  → key 失效广播
```

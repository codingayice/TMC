# 服务器基础设施说明

## 1. 服务器用途

该文档用于说明 TMC 项目所需的共享基础设施环境。实际部署时可以按“复用已有服务，补齐缺失服务”的方式准备 Redis、Kafka、etcd 和 rsyslog。

不要在文档中提交服务器登录密码或私钥。

## 2. 已复用服务

### 2.1 Redis

容器名：

```text
redis
```

用途：

- 作为 TMC 项目的集中缓存存储。

端口：

```text
宿主机 16379 -> 容器 6379
```

验证命令：

```bash
docker exec redis redis-cli ping
```

期望输出：

```text
PONG
```

### 2.2 Kafka

容器名：

```text
omini-kafka
```

用途：

- 作为 TMC 访问事件消息队列。

端口：

```text
宿主机 9092 -> 容器 9092
宿主机 9093 -> 容器 9093
```

TMC topic：

```text
tmc-access-events
```

验证命令：

```bash
docker exec omini-kafka sh -lc '(/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 --list) || (kafka-topics.sh --bootstrap-server localhost:9092 --list)'
```

期望包含：

```text
tmc-access-events
```

## 3. 新增服务

### 3.1 etcd

容器名：

```text
tmc-etcd
```

用途：

- 热点 key 快照下发。
- 本地缓存失效事件广播。

镜像：

```text
quay.io/coreos/etcd:v3.5.15
```

端口：

```text
宿主机 2379 -> 容器 2379
宿主机 2380 -> 容器 2380
```

数据卷：

```text
tmc-etcd-data
```

验证命令：

```bash
docker exec tmc-etcd etcdctl endpoint health --endpoints=http://127.0.0.1:2379
```

期望输出包含：

```text
is healthy
```

### 3.2 rsyslog

容器名：

```text
tmc-rsyslog
```

用途：

- 接收 SDK 写入的访问事件。
- 通过 `omkafka` 转发事件到 Kafka topic `tmc-access-events`。

镜像：

```text
tmc-rsyslog:latest
```

端口：

```text
宿主机 5514/tcp -> 容器 5514/tcp
宿主机 5514/udp -> 容器 5514/udp
```

配置路径：

```text
/opt/tmc/rsyslog/rsyslog.conf
```

日志路径：

```text
/opt/tmc/logs/rsyslog
```

## 4. TMC 事件链路验证

发送测试事件：

```bash
./scripts/smoke-infra.ps1
```

双服务器部署时使用远端模式：

```powershell
./scripts/smoke-infra.ps1 `
  -Mode Remote `
  -AppHost <app-host> `
  -AppUser <app-user> `
  -KafkaHost <kafka-host> `
  -KafkaUser <kafka-user> `
  -KafkaContainer <kafka-container>
```

期望输出包含：

```text
Infrastructure smoke check passed.
```

## 5. TMC 开发连接信息模板

后续 Java 项目中可以按实际部署环境填写以下连接配置：

```properties
tmc.redis.host=<redis-host>
tmc.redis.port=16379
tmc.kafka.bootstrap-servers=<kafka-host>:9092
tmc.etcd.endpoints=http://<etcd-host>:2379
tmc.rsyslog.host=<rsyslog-host>
tmc.rsyslog.port=5514
tmc.kafka.topic=tmc-access-events
```

## 6. 当前 TMC 相关容器

```text
tmc-rsyslog
tmc-etcd
omini-kafka
redis
```

其他已有服务如 MySQL、RabbitMQ、Portainer、pgvector 与 TMC Phase 1 无直接关系，不作为 TMC 链路依赖。
